## 2. Project Goal

Build a **mobile-first, book-style reading website** for the light novel "THE PILLAR." Readers can browse episodes, read content, and comment. Unregistered visitors see a **2-paragraph preview** of Episode 1 only — all other content requires a free account.

## 3. Core Features (locked)

### 3.1 Reading Gate (Paywall Logic)
```
Visitor arrives
  │
  ├─ Has Appwrite session? ──YES──► Full access to all episodes
  │
  └─ NO
       ├─ Episode 1 only is visible in sidebar
       ├─ Show first 2 paragraphs of Ep. 1
       ├─ Blur paragraph 3+ with CSS filter: blur(4px)
       └─ Inject gate card: "Create free account / Sign in"
            └─ On auth success ──► Full access, all episodes unlocked
```

- Episodes 2, 3, 4... show a 🔒 lock icon in the sidebar for guests
- Clicking a locked episode redirects to auth modal (does not navigate)
- Episode 1 is permanently free to preview (2 paragraphs), even after more episodes added

### 3.2 Language Selector
- Rendered as a `<select>` dropdown in the navbar (not a toggle)
- Options: 🇰🇷 한국어 · 🇺🇸 English (extensible — add more without redesign)
- Selected language stored in a cookie (`pillar_lang`, 365-day expiry)
- On change: navigates to the same episode slug in the other language
  - `/ko/ep1/` ↔ `/en/ep1/`
- Default: detect browser language, fall back to Korean

### 3.3 Visitor Counter
- Per-episode view count stored in Appwrite Database (`visitors` collection)
- Incremented via **Appwrite Function** (not client SDK directly — prevents abuse)
- Displayed in the right panel on desktop, below episode header on mobile
- Site-wide total shown on the homepage hero

### 3.4 Comment Section
- Guests can **read** all comments (no login required to view)
- Guests see a "Sign in to comment" prompt instead of the input box
- Logged-in users can post comments
- Comments are **language-scoped** — Korean comments show on `/ko/`, English on `/en/`
- Fields: `userId`, `episodeSlug`, `language`, `body`, `createdAt`
- No nested replies (flat list, newest first)
- Basic validation: body must be 3–500 characters

### 3.5 Auth (Email only)
- Signup: name + email + password
- Signin: email + password
- No OAuth, no magic links
- Appwrite handles sessions (cookie-based)
- Auth modal is a centered overlay, triggered by:
  - Clicking "Create account" / "Sign in" in navbar
  - Hitting the reading gate
  - Clicking a locked episode
- After successful auth: close modal, reload reader state (no full page reload)

## 4. Pages / Routes

## 6. Non-Negotiables

1. **No full page reload on auth** — modal closes, JS re-evaluates gate state
2. **Language persists across navigation** — cookie, not URL param
3. **Comments are language-scoped** — Korean readers see Korean comments
4. **Visitor counter goes through Appwrite Function** — not direct DB write
5. **CSS blur gate only** — episode content is in the HTML (static site); gate is UX friction, not DRM
6. **Mobile-first CSS** — write mobile styles first, use `@media (min-width: 768px)` for desktop enhancements
7. **No external UI libraries** — vanilla CSS only, no Tailwind, no Bootstrap
8. **Appwrite SDK loaded client-side** — 11ty has no server runtime

## 8. Environment Variables

Set these in Vercel dashboard AND in `.env.local` for local dev:

```env
APPWRITE_ENDPOINT=https://cloud.appwrite.io/v1
APPWRITE_PROJECT_ID=your_project_id
APPWRITE_DATABASE_ID=your_database_id
APPWRITE_COLLECTION_COMMENTS=comments
APPWRITE_COLLECTION_VISITORS=visitors
APPWRITE_FUNCTION_INCREMENT=increment_visitor
```

> These are build-time readable in 11ty for any static config. Appwrite client SDK reads `APPWRITE_ENDPOINT` and `APPWRITE_PROJECT_ID` at runtime — **never expose API keys to the client**.

## 10. Done Criteria

A build is considered complete when:

- [ ] Guest sees 2-paragraph preview of Ep. 1, gate card appears after
- [ ] Guest clicking Ep. 2–4 triggers auth modal, not navigation
- [ ] Language dropdown switches between `/ko/` and `/en/` routes
- [ ] Selected language persists in cookie across browser sessions
- [ ] Signup (email+password) creates Appwrite account and closes modal
- [ ] Signin authenticates and unlocks full reading access
- [ ] Visitor counter increments on episode page load
- [ ] Comments load for the current episode + language
- [ ] Logged-in users can post comments (3–500 chars)
- [ ] Mobile layout is single-column with no horizontal overflow
- [ ] Desktop layout shows 3-column (sidebar / reader / right panel)
- [ ] Vercel deployment builds without errors from `_site/`
